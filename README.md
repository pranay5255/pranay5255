## üìö Research Fueling My Builds

I read to ship. These are the ideas currently shaping **Yudai v3**, **solo-server**, and my **local-first agent stack**:

### Reasoning, RL, and verifiable rewards
- **[DeepSeek-R1](https://arxiv.org/abs/2501.12948)** ‚Äî RL + distillation patterns for efficient reasoning (and what "reasoning supervision" really looks like)
- **[Reinforcement Learning with Verifiable Rewards](https://arxiv.org/abs/2503.23829)** ‚Äî reward design where correctness is checkable, not vibes
- **[Absolute Zero Reasoner](https://arxiv.org/abs/2505.03335)** ‚Äî pushing capability with minimal scaffolding + strong evaluation discipline
- **DeepSeek-R1 distillation notes** ‚Äî how to compress "thinking" into smaller codex-grade models without losing usefulness

### SWE agents, debugging, and data engines for code
- **[SWE-smith](https://arxiv.org/abs/2504.21798)** ‚Äî synthetic bug/task generation as a scaling lever for SWE agents
- **[debug-gym](https://arxiv.org/abs/2503.21557)** ‚Äî tool-augmented debugging environments + structured feedback loops
- **[LLM ‚Üí SLM Agent Conversion](https://arxiv.org/pdf/2506.02153)** ‚Äî turning general LLMs into specialist codex/agent SLMs for workflows
- **SWE-agent / mini-SWE patterns (execution harnesses)** ‚Äî repo parsing ‚Üí plan ‚Üí edit loop ‚Üí tests/CI ‚Üí PR (the stuff that actually ships)

### Local-first models and efficient systems
- **[Qwen3 Technical Report](https://arxiv.org/abs/2505.09388)** ‚Äî model family tradeoffs + what makes Qwen great for local coders
- **Flash attention + long context scaling** ‚Äî the practical path: throughput first, then context, then agent reliability
- **[FlashAttention-3](https://arxiv.org/abs/2407.08608)** ‚Äî exact attention engineering for real speed (H100-class kernels)
- **[The Ultra-Scale Playbook (Hugging Face)](https://huggingface.co/spaces/nanotron/ultrascale-playbook/blob/main/The_Ultra-Scale_Playbook_Training_LLMs_on_GPU_Clusters.pdf)** ‚Äî cluster + org patterns that actually work
- **[Smol LM Training Playbook (Hugging Face)](https://huggingface.co/spaces/HuggingFaceTB/smol-training-playbook)** ‚Äî small-model recipes: KD, data curation, eval discipline, deployment constraints

### Tabular + product analytics foundations (for "AI analyst" workflows)
- **[Accurate predictions on small data with a tabular foundation model (Nature, 2025)](https://www.nature.com/articles/s41586-024-08328-6)** ‚Äî TabPFN-style few-shot tabular learning (small data, real impact)

### Systems worldview + where AI is pushing the frontier
- **[Barbarians at the Gate: How AI is Upending Systems Research](https://arxiv.org/abs/2510.06189)** ‚Äî how research changes when agents can prototype systems rapidly

### Crypto mechanism design (because I can't not üòÑ)
- **[NoFeeSwap Yellow Paper](https://www.nofeeswap.org/yellowpaper.pdf)** ‚Äî AMM design + liquidity math (I'm prototyping in Solidity)

> Current obsession: **specialized SLM codex agents** + **verifiable reward loops** + **local inference** + **reliable PR shipping**.
